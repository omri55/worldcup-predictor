# ⚽ חיזוי מונדיאל 2026

מערכת חיזוי למשחקי מונדיאל 2026 — חינמית לחלוטין, מתעדכנת אוטומטית, מבוססת ניתוח סטטיסטי עמוק.

נותנת לכל משחק: הסתברויות 1X2, תוצאה מומלצת, xG צפוי, התפלגות תוצאות מדויקות, over/under,
"שתי הקבוצות יבקיעו", ורמת ביטחון. מושלם למשחקי ניחושים עם חברים.

## איך זה עובד

| רכיב | תיאור |
|------|-------|
| **נתונים** | [martj42/international_results](https://github.com/martj42/international_results) — כל תוצאות הנבחרות מ-1872, כולל לוח מונדיאל 2026. חינם (CC0), מתעדכן רציף. |
| **מודל גולים** | **Dixon-Coles** (Poisson דו-משתני) — חוזה התפלגות גולים מלאה לפי כוח התקפה/הגנה לכל נבחרת, עם תיקון לתוצאות נמוכות ומשקל לפי עדכניות (חצי-חיים שנתיים). |
| **דירוג כוח** | **Elo** בנוסח eloratings.net — חישוב רציף מכל ההיסטוריה, עם משקל לפי חשיבות הטורניר והפרש שערים. |
| **שוק ההימורים** | [The Odds API](https://the-odds-api.com) (tier חינמי) — יחסי 1X2 מ-~24 בוקמייקרים (כולל Pinnacle). מנקה את עמלת הבוקמייקר (vig) ומחשב הסתברות שוק "אמיתית". אות חזק שמגלם פציעות/הרכבים/חדשות. |
| **שילוב** | כשיש יחסי שוק: 50% שוק + 50% (65% Dixon-Coles, 35% Elo). אחרת — רק המודל הסטטיסטי. |
| **עדכון** | רענון אוטומטי כל 12 שעות (נתונים + יחסים + אימון מחדש) — תמיד עדכני. צריכת API: ~60-120 בקשות/חודש מתוך 500. |

**דיוק נוכחי:** ~62% פגיעה בתוצאת 1X2 על משחקי המונדיאל שכבר שוחקו.

## הרצה

### Backend (Python / FastAPI)
```bash
cd backend
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
./.venv/bin/uvicorn app.main:app --reload --port 8000
```
האימון הראשוני בעלייה לוקח כ-90 שניות. API docs: http://127.0.0.1:8000/docs

### Frontend (React / Vite)
```bash
cd frontend
npm install
npm run dev
```
פתח http://localhost:5173 (ה-frontend עושה proxy ל-backend על פורט 8000).

## סימולציית מונטה-קרלו (`models/simulator.py`)
מריץ אלפי ריצות מלאות של הטורניר: גוזר את 12 הבתים אוטומטית ממבנה המשחקים,
משחק את שלב הבתים (כולל **תוצאות אמת** של משחקים שכבר נגמרו), מדרג עם שוברי
השוויון הרשמיים (נקודות → הפרש שערים → שערי זכות) ומעפיל 2 מכל בית + 8
המקומות-שלישיים הטובים. שלב הנוקאאוט משוחק כעץ חד-הדפי בזירוע לפי Elo.
התוצאה: הסתברות העפלה / רבע / חצי / גמר / זכייה לכל נבחרת.
> הערה: זירוע הנוקאאוט הוא קירוב מבוסס-כוח עד שמשחקי שלב הנוקאאוט הרשמיים
> ייכנסו לנתוני המקור — אז אפשר לחבר את הצמדות הבית האמיתיות.

## הגדרת מפתח The Odds API
מפתח חינמי מ-https://the-odds-api.com נשמר ב-`backend/.env` (מוגן ב-gitignore):
```
ODDS_API_KEY=your_key_here
```
בלי מפתח — המערכת עובדת רגיל על המודל הסטטיסטי בלבד (שכבת השוק פשוט מושבתת).

## מנוע ניחושים (תוחלת נקודות)
ללוטו/ניחושים עם חברים, ה"ניחוש המומלץ" הוא **לא** התוצאה הכי סבירה אלא זו עם
**תוחלת הנקודות הגבוהה ביותר** לפי שיטת הניקוד של הליגה (ב-`config.prediction_scoring`,
מדורגת לפי שלב). המנוע משקלל מחדש את מטריצת התוצאות של Dixon-Coles כך שתואמת
ל-1X2 המשולב, ובוחר את `argmax( exact_pts·P(score) + dir_pts·(P(direction)−P(score)) )`.
פרמטר `stage` ב-`/api/predict` (group/r32/r16/qf/sf/third/final) קובע את הניקוד.

בקטסט אאוט-אוף-סמפל: `python backtest.py` (אימון עד 2024, בדיקה על ~2,552 משחקים).

## רעיונות להמשך
- שילוב יחסי השוק גם בסימולציית מונטה-קרלו (כרגע מבוססת גולים בלבד).

- חיבור הצמדות הנוקאאוט הרשמיות כשייכנסו לנתונים.
- טבלת ניחושים שיתופית לחברים עם ניקוד.
